How should connected users appear in the terminal view?

This is the session viewer page (/app/session/{id}) — where multiple people watch the same terminal

A

Avatar Bar (top/bottom strip)

A row of colored circles with initials above or below the terminal. Shows who's connected, their role badge (viewer/operator/admin), and an idle indicator. Clicking an avatar shows their details.

B

Sidebar Panel (collapsible)

A narrow panel on the right showing connected users as a vertical list. Each entry shows name, role, connection time, and activity status. Panel collapses to icons when terminal needs space.

C

Overlay Cursors (Google Docs style)

Each user gets a colored cursor label that appears on the terminal at their "focus position" (last line they scrolled to). No separate UI panel — presence is shown directly in the terminal area. Minimal, but powerful for pair work.

$ kubectl get pods -n production
NAME                READY   STATUS
api-server-7d4f   1/1     Running Jane
worker-batch-9c   0/1     CrashLoopBackOff Tim
D

Avatar Bar + Overlay Cursors (A + C combined)

Avatar bar for who's connected (always visible), plus optional cursor overlays in the terminal for focus tracking. Best of both worlds. Cursors can be toggled off if distracting.
