Terminal bridge pattern

The vanilla UndefTerminal class manages its own DOM, xterm.js instance, and WebSocket connection. React can't own that lifecycle — it needs to host it.

// terminal/useTerminalRef.ts
const containerRef = useRef<HTMLDivElement>(null)
const termRef = useRef<UndefTerminal | null>(null)
useEffect(() => {
if (!containerRef.current) return
termRef.current = new UndefTerminal(
containerRef.current,
{ wsUrl, theme, cols, rows }
)
return () => termRef.current?.dispose()
}, [wsUrl])

The React component provides the container div and config; the vanilla class owns everything inside it. getBufferText() bridges screen content back to React for the analyze feature. No React-xterm wrapper library needed.
